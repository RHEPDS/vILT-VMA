# vILT-VMA — Virtualization Migration Assessment (Technical Workshop)

Virtual instructor-led training materials and context for the **Virtual Migration Assessment (VMA)** technical workshop: positioning Red Hat **OpenShift Virtualization** as the path from VMware-based infrastructure, and running assessments with partners.

**Related:** [Google Doc (brief)](https://docs.google.com/document/d/1V_kI1_BhjS9CEbnLH5A5qRt0zvSi_-xQzYdcMttv3C8/edit?usp=sharing) · Source export: `VMA_vILT_2026.rtf`

## Business problem

There is a unique market opportunity for Red Hat to support customers moving from VMware-based infrastructure to an OpenShift-based stack, in light of recent changes under VMware’s ownership (Broadcom). A core activity to drive this motion is performing a **Red Hat Virtualization Migration Assessment**. Today, there are not enough technical sellers who can consistently position the offer and secure customer commitment to take the first step away from VMware.

## Audience

- Technical sales / presales  
- Business consultants  
- Delivery consultants  

## Prerequisites

- OpenShift knowledge (e.g. DO180)  
- Basic understanding of OpenShift Virtualization  
- VMware knowledge  

## Workshop goal

Build confidence to **identify and convince** customers to commit to a Virtualization Migration Assessment (partner delivery guide), including:

- Understanding the customer’s existing environment  
- Assessing migration risk  
- Running an assessment **as a partner on the customer side**  

## Intended outcomes

Participants should be able to:

- Identify and qualify opportunities for a Virtualization Migration Assessment  
- Position the business and technical value of the assessment  
- Handle technical or business competitive situations appropriately  
- Know when and how to involve an ecosystem partner for an assessment  
- Perform upfront technical discovery and risk assessment  
- Scope a proof of concept for OpenShift Virtualization  
- Communicate technical value during delivery of the virtualization roadshow
